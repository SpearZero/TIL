
### Clean Code
- 경계, 단위 테스트, 클래스, 시스템
    - 경계 : 외부 코드를 사용해 프로그램을 작성할 때의 주의점
    - 단위 테스트 : 테스트 코드는 실제 코드의 유연성, 유지보수성, 재사용성을 보존하고 강화
    - 클래스 : 클래스는 작고, 변경하기 쉽게 만들어야 한다.
    - 시스템 : 시스템을 쉽게 확장하기 위해서는 관심사를 분리하고 잘 관리해야 한다.
