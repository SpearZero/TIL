## 시작하며: 바이브 코딩이란 무엇인가?

- 바이브 코딩
    - 사용자가 바라는 기능을 설명하는 프롬프트를 작성하면, 대규모 언어 모델(LLM)이 남은 부분을 채워나가는 코딩 방식

### 바이브 코딩과 AI 보조 엔지니어링
- 바이브 코딩
    - AI가 코드 생성을 주도하고, 사용자는 높은 수준의 목표에 집중
    - 초기 설정 작업을 생략하고, 구현을 프롬프트로 빠르게 완성
    - 적합 대상
        - 데모 제작자
        - 프로덕트 지향 개발자
        - 실무 경험이 부족한 신입 개발자
    - 주의점
        - 설계 품질 저하 및 잠재적 버그 위험
        - 결과물은 완성본이 아닌 출발점
- AI 보조 엔지니어링
    - 개발자가 개발 과정을 주도하고, AI는 조력자 역할
    - 계획 → AI 코드 생성 → 검토 → 반복 흐름 (코드 생성 -> 검토 -> 프롬프트 입력 반복)
    - 프롬프트보다 계획과 설계가 우선
        - 요구사항 문서
        - 상세 명세서
        - 작업 항목 체크리스트
    - 목표는 빠른 구현이 아닌 고품질 코드의 효율적 작성

바이브 코딩과 AI 보조 프로그래밍의 목표
- 바이브 코딩
    - 단기적인 속도에 집중
- AI 보조 엔지니어링
    - 지속 가능한 속도와 신뢰성에 집중

바이브 코딩과 AI 보조 엔지니어링을 시기적절하게 사용하는것이 좋음

### 의도 중심 프로그래밍
- 세부 구현보다 프로그램이 달성해야 할 목적(What)에 집중한다.
- 개발자는 어떻게 구현할지보다 무엇을 만들지를 정의한다.
- AI는 목적을 바탕으로 구현 방법(How)을 탐색한다.
- 프롬프트
    - 실행 절차가 아닌 기능과 의도 중심으로 작성
    - 의도가 명확할수록 AI가 생성하는 코드의 품질이 높아진다.

AI 작동 흐름
- 프롬프트 이해 → 컨텍스트 활용 → 코드 생성 → 인간 감독을 통한 검증

바이브 코딩은 인간 – AI 간 반복적 협업을 전제로 한다.
- 한 번의 프롬프트로 모든 구현을 맡기는 방식은 바람직하지 않다.
- 반복적 협업의 과정
    - 의도를 설명 → AI가 초기 코드 제안 → 코드 리뷰와 테스트 진행 → AI 에게 다시 요청하거나 직접 코드를 개선 → AI가 솔루션을 개선 → 필요한 만큼 반복
- 반복의 목적
    - 오류의 수정이 아닌 솔루션의 발전

### 의도 중심 프로그래밍의 중요성과 어려움
- 중요성
    - 개발자의 생산성 향상
        - 반복적, 기계적인 작업을 AI에 위임
    - 개발자의 집중력 유지
        - 기본 코드 작성과 문법 부담에서 벗어나 문제 해결에 집중
    - 진입 장벽 축소
        - 코딩 지식이 깊지 않아도 프로그래밍에 더 쉽게 접근 가능
    - 개발자의 역할과 기술의 변화
        - 구현보다 아키텍처 설계, 문제 분해, 검증 능력이 중요해짐
    - 생상선 vs 창의성
        -  생산적인 일은 AI에게, 창의적인 일은 개발자가 담당
- 어려움
    - 신뢰와 정확성
        - AI를 무작정 믿는 건 위험하다.
    - 저수준 기술의 망각
        - 개발자는 편리함과 기본기의 균형을 의식적으로 유지해야 한다.
    - 변화하는 취업 시장
        - 개발자에 대한 수요는 줄고, 시스템 설계, 컴포넌트 통합, 정확성 검증 같은 능력을 가진 개발자를 찾는 수요가 늘어날 것이다.

### AI 모델 범주
모델 범주
- 속도 최적화
    - 빠른 응답과 반복에 최적화
    - 프로토타이핑, 일반 코딩 작업에 적합
    - 복잡한 문제 해결에는 한계
- 심층 추론
    - 느리지만 복잡한 문제를 깊이 있게 해결
    - 복잡한 디버깅, 시스템 설계에 적합
- 멀티모달 시스템 
    - 이미지, 도표, 비디오 콘텐츠 처리
- 오픈소스 모델
    -  무료 사용 가능
    - 기능 및 품질에 제약이 있을 수 있음

모든 모델을 위한 실용적인 팁
- 컨텍스트를 충분히 제공한다
- 생성된 결과를 검증하고 개선을 요청한다
    - 문제점이나 이유 설명을 요구하면 품질이 향상된다
- 컨텍스트 유지 능력을 활용한다
    -  초기 구현 후, 반복 요청을 통해 점진적으로 개선

### 바이브 코딩과 이점의 한계
이상적인 사례
- 제로 투 원 개발
    - 프로토타입을 빠르게 구현
- 단순 기능 및 CRUD 개발
    - 복잡한 비즈니스 논리, 데이터 검증 규칙, 기존 시스템과의 통합이 추가되면 AI 보조 엔지니어링 방식이 중요해진다.
- 코드 통합 및 반복 작업
- 현대 프레임워크 활용
    - 주요 프레임워크 매뉴얼을 기반으로 한 빠른 구현

AI 보조 엔지니어링이 우선인 경우
- 정교한 데이터 구조 설계
- 성능이 중요한 알고리즘
- 새로운 계산 문제 해결
- 조직의 핵심 비즈니스 로직 구현

빠른 실험과 치밀한 설계 사이를 상황에 맞게 전환하는 것이 AI를 제대로 활용하는 방법이다.

### AI가 여전히 어려움을 겪는 영역
- 매우 복잡한 시스템
- 저수준 최적화 및 시스템 프로그래밍
- 독특하거나 생소한 프레임워크
- 창의적인 UI/UX 디자인
- 의도와 요구사항 해석
    - 요구사항이 암묵적이거나 모순적일 떄 어려움을 겪는다.

AI 보조 엔지니어링의 장점
- 더 빠른 개발 주기
- 프로토타입 제작과 실험의 강화
    - 낮은 비용으로 다양한 시도 가능
- 손끝에 닿는 지식
    - 생각하지 못한 해결책이나 아이디어 제공
- 일관성과 표준화
    - 코딩 규칙과 모범 사례 준수 지원

고려 사항과 트레이드 오프
- 변수 출력 품질
    - 미세한 버그나 비효율 가능성
- 코드를 모호하게 만드는 프롬프트의 모호함
- 과도한 의존에 따른 기술 퇴화
- 프라이버시와 보안 문제
- AI 결과의 편향
- 인간적 요소와 신뢰
    - 팀 단위 도입 시 적응 기간과 경험 공유 필요
