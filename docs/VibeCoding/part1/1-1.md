## 시작하며: 바이브 코딩이란 무엇인가?

- 바이브 코딩
    - 사용자가 바라는 기능을 설명하는 프롬프트를 작성하면, 대규모 언어 모델(LLM)이 남은 부분을 채워나가는 코딩 방식

### 바이브 코딩과 AI 보조 엔지니어링
- 바이브 코딩
    - AI가 코드 생성을 주도하고, 사용자는 높은 수준의 목표에 집중
    - 초기 설정 작업을 생략하고, 구현을 프롬프트로 빠르게 완성
    - 적합 대상
        - 데모 제작자
        - 프로덕트 지향 개발자
        - 실무 경험이 부족한 신입 개발자
    - 주의점
        - 설계 품질 저하 및 잠재적 버그 위험
        - 결과물은 완성본이 아닌 출발점
- AI 보조 엔지니어링
    - 개발자가 개발 과정을 주도하고, AI는 조력자 역할
    - 계획 → AI 코드 생성 → 검토 → 반복 흐름 (코드 생성 -> 검토 -> 프롬프트 입력 반복)
    - 프롬프트보다 계획과 설계가 우선
        - 요구사항 문서
        - 상세 명세서
        - 작업 항목 체크리스트
    - 목표는 빠른 구현이 아닌 고품질 코드의 효율적 작성

바이브 코딩과 AI 보조 프로그래밍의 목표
- 바이브 코딩
    - 단기적인 속도에 집중
- AI 보조 엔지니어링
    - 지속 가능한 속도와 신뢰성에 집중

바이브 코딩과 AI 보조 엔지니어링을 시기적절하게 사용하는것이 좋음

### 의도 중심 프로그래밍
- 세부 구현보다 프로그램이 달성해야 할 목적(What)에 집중한다.
- 개발자는 어떻게 구현할지보다 무엇을 만들지를 정의한다.
- AI는 목적을 바탕으로 구현 방법(How)을 탐색한다.
- 프롬프트
    - 실행 절차가 아닌 기능과 의도 중심으로 작성
    - 의도가 명확할수록 AI가 생성하는 코드의 품질이 높아진다.

AI 작동 흐름
- 프롬프트 이해 → 컨텍스트 활용 → 코드 생성 → 인간 감독을 통한 검증

바이브 코딩은 인간 – AI 간 반복적 협업을 전제로 한다.
- 한 번의 프롬프트로 모든 구현을 맡기는 방식은 바람직하지 않다.
- 반복적 협업의 과정
    - 의도를 설명 → AI가 초기 코드 제안 → 코드 리뷰와 테스트 진행 → AI 에게 다시 요청하거나 직접 코드를 개선 → AI가 솔루션을 개선 → 필요한 만큼 반복
- 반복의 목적
    - 오류의 수정이 아닌 솔루션의 발전

### 의도 중심 프로그래밍의 중요성과 어려움
- 중요성
    - 개발자의 생산성 향상
        - 반복적, 기계적인 작업을 AI에 위임
    - 개발자의 집중력 유지
        - 기본 코드 작성과 문법 부담에서 벗어나 문제 해결에 집중
    - 진입 장벽 축소
        - 코딩 지식이 깊지 않아도 프로그래밍에 더 쉽게 접근 가능
    - 개발자의 역할과 기술의 변화
        - 구현보다 아키텍처 설계, 문제 분해, 검증 능력이 중요해짐
    - 생상선 vs 창의성
        -  생산적인 일은 AI에게, 창의적인 일은 개발자가 담당
- 어려움
    - 신뢰와 정확성
        - AI를 무작정 믿는 건 위험하다.
    - 저수준 기술의 망각
        - 개발자는 편리함과 기본기의 균형을 의식적으로 유지해야 한다.
    - 변화하는 취업 시장
        - 개발자에 대한 수요는 줄고, 시스템 설계, 컴포넌트 통합, 정확성 검증 같은 능력을 가진 개발자를 찾는 수요가 늘어날 것이다.
