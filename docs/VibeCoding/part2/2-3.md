## 70% 문제: 효과적인 AI 보조 워크플로

- AI는 개발자를 강화하는 도구로, 반복적인 작업의 약 70%를 대신 처리하여 운영 효율을 높인다.
- 약 30%의 일(엣지 케이스, 유지보수, 견고한 아키텍처 구축)은 개발자의 전문적인 역량이 필요하다.

AI가 기능을 빠르게 구현하지만, 개발자의 전문성이 코드를 안정적으로 유지하는 데 중요한 역할을 한다.

AI 보조 워크플로 안티패턴
- 제자리 걸음
    - 작은 버그 발생 → AI에게 요청 → 수정으로 또 다른 문제 발생 → 다시 AI에게 요청 → 동일한 패턴이 반복되는 상태
    - 기본기를 소홀히 하고 AI에만 의존하면, AI가 잘못된 방향으로 갈 때 이를 검증하거나 바로잡기 어렵다.
    - AI 도구를 효과적으로 사용하기 위해서는 소프트웨어의 본질과 문제의 원인을 깊이 이해해야 한다.
- 데모 품질의 함정
    - AI를 활용하면 데모를 빠르게 제작할 수 있지만, 실제 사용 단계에서는 다양한 문제가 드러날 수 있다.
        - 엣지 케이스 미처리
        - 혼란스러운 UI
        - 접근성 미고려
        - 성능 문제
    - 완성도 있는 결과물은 자동화가 아니라 세심한 검토와 마무리에서 나온다.

실용적인 AI 보조 워크플로 패턴
- AI의 초안 작성
    - 팀 차원에서 코딩 표준과 프롬프트 작성 방식을 합의한다.
    - AI 세션이나 프롬프트를 공유해 작업 맥락을 유지한다.
    - AI 보조 코딩 시 자주 커밋하는 전략이 유용하다.
        - 변경 사항을 되돌리기 쉽다.
    - 커밋 메시지에 AI 작업 여부를 명시해 기록 신뢰성을 높인다.
    - AI가 생성한 결과물은 반드시 꼼꼼히 검토하고 개선한다.
- AI와 페어 프로그래밍
    - AI가 코드 초안을 제안하고, 개발자가 이를 검토·개선하는 방식
    - 모범 사례
        - 작업 단위로 새로운 AI 세션을 시작한다.
        - 명확하고 간결한 프롬프트를 사용한다.
        - 잦은 확인과 커밋을 통해 변경 이력을 관리한다.
        - 지속적인 피드백 루프를 유지한다.
- AI를 사용한 검증
    - AI는 코드 리뷰, 테스트 생성, 보안·성능 분석을 통해 개발 과정에서 결함을 조기에 발견하고 품질을 높이는 검증 도구로 활용된다.

바이브 코딩의 핵심 원칙
- 의도를 명확히 전달하고, 결과를 검증한다
    - 원하는 내용을 구체적으로 전달한다.
    - AI의 출력이 원래 의도와 일치하는지 항상 확인한다.
- AI는 조력자이며, 책임은 인간에게 있다
  - AI를 주니어 개발자처럼 감독한다.
  - 사고를 AI에 위임하지 않고, 자신의 역량을 확장하는 도구로 활용한다.
  - 이해하지 못하는 코드는 머지하지 않는다.
- 규칙과 기준을 먼저 정한다
  - 코드를 생성하기 전에 규칙과 제약을 명확히 한다.
  - 인간이 작성했든 AI가 작성했든 모든 코드는 동일한 리뷰 기준을 적용한다.
- 변경 이력을 명확히 관리한다
    - AI가 생성한 변경 사항은 분리된 커밋으로 관리한다.
    - 리뷰와 롤백이 가능하도록 변경 단위를 작게 유지한다.
- 지식과 맥락을 우선시한다
    - 문서, 주석, 아키텍처 결정 기록(ADR)을 중시한다.
    - 효과적인 프롬프트를 공유하고 재사용한다.
- 지속적으로 개선한다
    - AI 사용 방식을 개발 문화의 자연스러운 일부로 받아들인다.
    - 정기적으로 작업 방식을 돌아보고 반복적으로 개선한다.
